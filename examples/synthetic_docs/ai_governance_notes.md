# Synthetic AI Governance Notes

These notes summarize representative governance practices for AI systems using synthetic content.

## Evaluation Governance

Evaluation suites should include factuality, grounding, safety, privacy, latency, and robustness checks. Results should be versioned so reviewers can compare changes over time.

## Human Review

High-risk failures require review by an accountable owner. Automated checks should create review queues when answers are unsupported, unsafe, or inconsistent with required output format.

## Risk Register

Known risks include prompt injection, private data leakage, stale retrieval sources, model extraction, membership inference, and overreliance on uncalibrated model judgments.
